<h2 id="section-1-1-1">🎯 1.1.1 项目背景</h2>

### TRQuant的诞生

TRQuant（韬睿量化）是**开发团队内部使用的投资辅助工具**，面向A股市场，旨在实现投资流程的**自动化、智能化、可视化**，为团队投资决策提供系统化支持。

### 项目定位

**核心定位**：内部投资辅助工具

- **用户群体**：开发团队内部成员
- **使用场景**：辅助投资决策，提升投资效率和质量
- **发展策略**：当前专注于内部使用，后期再考虑对外推广

### 项目背景

在团队投资实践中，面临以下挑战：

1. **工具碎片化**：数据获取、因子计算、策略开发、回测验证分散在不同工具中，使用不便
2. **流程不连贯**：各环节缺乏统一的工作流，难以形成完整的投资闭环
3. **知识难复用**：策略知识、开发经验难以积累和复用，重复劳动多
4. **效率待提升**：手动操作多，缺乏自动化工具，影响投资效率

### 核心价值主张

TRQuant致力于解决团队内部投资实践中的痛点，提供：

- **一体化系统**：从数据获取到实盘交易的全链路支持，统一工具平台
- **自动化工作流**：8步骤投资流程自动化执行，减少手动操作
- **智能化辅助**：AI技术全面应用，辅助投资决策，提升判断准确性
- **可视化界面**：桌面系统GUI和Cursor扩展，直观易用，提升使用体验

### 项目基调

**重要说明**：

- **内部工具定位**：TRQuant是开发团队内部使用的投资辅助工具，不是面向外部用户的产品
- **辅助投资目标**：系统旨在辅助团队投资决策，提升投资效率和质量
- **发展策略**：当前阶段专注于内部使用和功能完善，后期再考虑对外推广
- **实用导向**：所有功能设计都围绕实际投资需求，注重实用性和可落地性

<h2 id="section-1-1-2">🔍 1.1.2 核心定位</h2>

### TRQuant ≠ 简单的回测平台

TRQuant与传统回测平台存在本质区别：

| 特性 | 传统回测平台 | TRQuant |
|------|------------|---------|
| **核心定位** | 回测平台 | 完整投资流程系统 |
| **工作流程** | 策略→回测→结果 | 8步骤完整工作流 |
| **启动界面** | 项目管理 | 投资工作流仪表盘 |
| **目标市场** | 美股为主 | A股专属 |
| **券商对接** | Interactive Brokers | PTrade/QMT |
| **数据源** | 单一数据源 | 统一数据源管理 |
| **策略生成** | 手动编写 | 基于Strategy KB自动生成 |
| **知识积累** | 无 | RAG知识库 + Strategy KB |
| **工具链** | 无 | MCP Server + Engineering Server |

### 完整投资流程系统

TRQuant涵盖投资流程的各个环节：

```
📡 信息获取 → 📈 市场趋势 → 🔥 投资主线 → 📦 候选池构建 
  → 📊 因子构建 → 🛠️ 策略生成 → ⚡ 策略优化 → 🔄 回测验证 → 🚀 实盘交易
```

每个环节都有对应的模块支持，形成完整的闭环。

<h2 id="section-1-1-3">🎯 1.1.3 系统目标</h2>

### 总体目标

实现一个**可落地、可复现、可审计**的量化投资系统，为前五册投资手册提供工程化能力支撑。

### 三大核心目标

#### 1. 自动化 - 完整投资流程自动化

**8步骤工作流自动执行**：

```
📡 信息获取 → 📈 市场趋势 → 🔥 投资主线 → 📦 候选池构建 
  → 📊 因子构建 → 🛠️ 策略生成 → ⚡ 策略优化 → 🔄 回测验证 → 🚀 实盘交易
```

**具体实现**：

- **数据自动获取和处理**：统一数据源管理，自动数据质量检查
- **策略自动生成和优化**：基于Strategy KB自动生成策略，自动优化迭代
- **实盘自动交易和监控**：PTrade/QMT集成，自动交易执行和风险监控

#### 2. 智能化 - AI技术全面应用

**AI能力覆盖**：

- **AI市场分析**：市场状态判断（risk_on/risk_off/neutral）、趋势预测
- **AI主线识别**：投资主线评分和筛选，基于多维度数据综合分析
- **AI因子推荐**：基于市场状态的因子推荐，智能权重配置
- **AI策略生成**：基于Strategy KB的策略生成，策略逻辑优化
- **AI策略优化**：智能参数调优，优化算法选择

**技术支撑**：

- **RAG知识库**：Manual KB + Engineering KB，智能检索与生成
- **Strategy KB**：策略知识库，策略生成支持
- **MCP Server**：工程编排、文档管理、代码分析

#### 3. 可视化 - 全方位可视化展示

**可视化界面**：

- **桌面系统GUI**（PyQt6）：完整的工作流界面，实时进度显示
- **Cursor扩展界面**（TypeScript + React）：集成到IDE的量化工作台
- **工作流仪表盘**：实时进度、步骤状态、数据可视化
- **数据可视化**：图表、仪表盘、对比分析

### 具体目标

1. **工程化能力**：为前五册提供可落地的工程化能力
   - 数据源管理、因子库、策略开发、回测验证、平台集成

2. **工具链支撑**：提供完整的开发工具链
   - Engineering Server、RAG知识库、Strategy KB、MCP Server

3. **系统集成**：支持多种使用方式
   - 桌面系统（PyQt6 GUI）
   - Cursor扩展（IDE集成）
   - 命令行工具

4. **可追溯性**：所有操作可追溯、可复现、可审计
   - 版本管理、证据记录、变更追踪

<h2 id="section-1-1-4">👥 1.1.4 目标用户</h2>

### 主要用户群体（内部团队）

TRQuant是**开发团队内部使用的投资辅助工具**，主要面向以下内部用户：

1. **团队开发者**：需要快速开发、测试、优化策略
   - 需求：高效的开发工具、完整的回测框架、策略优化能力
   - 价值：提高开发效率，缩短策略迭代周期
   - 使用场景：策略开发、代码生成、回测验证

2. **团队投资人员**：需要市场分析、主线识别、因子研究
   - 需求：市场分析工具、主线识别能力、因子研究平台
   - 价值：提升研究效率，增强分析能力，辅助投资决策
   - 使用场景：市场分析、主线识别、因子研究、投资决策

3. **系统维护人员**：需要维护和优化系统功能
   - 需求：系统监控、性能优化、功能扩展
   - 价值：确保系统稳定运行，持续改进功能
   - 使用场景：系统维护、功能优化、问题排查

### 使用场景（内部实践）

1. **策略开发**：从市场分析到策略生成的完整流程
   - 使用工具：市场分析模块、主线识别模块、策略生成模块
   - 工作流：信息获取→市场趋势→投资主线→策略生成
   - 目标：快速生成符合团队投资理念的策略代码

2. **策略优化**：基于回测结果的策略迭代优化
   - 使用工具：回测模块、优化模块、对比分析
   - 工作流：回测验证→策略优化→参数调优
   - 目标：持续改进策略表现，提升投资效果

3. **实盘交易**：策略部署到PTrade/QMT进行实盘交易
   - 使用工具：平台集成模块、交易执行、风险监控
   - 工作流：策略部署→实盘交易→监控反馈
   - 目标：安全、稳定地执行投资策略

4. **研究分析**：市场分析、主线识别、因子研究
   - 使用工具：市场分析模块、主线识别模块、因子库
   - 工作流：数据获取→市场分析→主线识别→因子研究
   - 目标：深入理解市场，发现投资机会

### 未来推广考虑

**当前阶段**：专注于内部使用和功能完善

**后期考虑**：在系统成熟、功能完善后，再考虑对外推广的可能性

<h2 id="section-1-1-5">💎 1.1.5 项目价值</h2>

### 对用户的价值

1. **提高效率**：自动化工作流，减少重复劳动
   - 8步骤工作流自动执行，无需手动操作
   - 数据自动获取和处理，节省时间

2. **降低门槛**：可视化界面，无需深入编程
   - 桌面系统GUI，直观易用
   - Cursor扩展，IDE集成，开发友好

3. **提升质量**：AI辅助决策，智能优化建议
   - AI市场分析，提升判断准确性
   - AI策略生成，优化策略逻辑

4. **保证可追溯**：完整的证据链，可审计可复现
   - 所有操作记录证据
   - 版本管理，可追溯变更历史

### 对系统的价值

1. **知识积累**：Strategy KB、RAG知识库持续积累
   - 策略知识库，策略模式积累
   - RAG知识库，开发经验积累

2. **能力复用**：模块化设计，能力可复用
   - 统一API接口，易于集成
   - 模块化设计，功能可复用

3. **持续改进**：基于反馈的持续优化
   - 实盘反馈，策略优化
   - 用户反馈，系统改进

4. **生态建设**：为量化投资生态提供基础设施
   - 标准化接口，促进生态发展
   - 工具链支持，降低开发门槛

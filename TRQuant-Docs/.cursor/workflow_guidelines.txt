TRQuant-Docs 工作区工作流规范
============================

核心原则
--------
Composer 完全禁用，只使用 Chat 模式

Composer 使用规则
-----------------
❌ 完全禁用 Composer

原因：
- 文档文件通常很大（>1000行）
- Markdown 文件的 diff 渲染复杂
- Composer 处理大文档容易超时

标准流程
--------

文档生成流程
------------
1. 使用 Chat 模式
2. 让 AI 生成完整文档内容
3. 手动新建/覆盖文件
4. 保存文件
5. 使用 Git 管理版本

文档修改流程
------------
1. 使用 Chat 模式
2. 说明需要修改的内容
3. 复制 AI 生成的内容
4. 手动编辑文件
5. 保存文件

绝对禁止
--------
❌ 让 Composer 生成文档
❌ 让 Composer 编辑大文档（>100行）
❌ 让 Composer 批量修改文档
❌ 在 Composer 中一次性生成多个文档

文件大小建议
------------
- 单个文档建议 < 1000 行
- 大文档拆分为多个小文档
- 使用索引文件链接相关文档

最佳实践
--------
1. 文档拆分：大文档拆分为多个小文档
2. 版本管理：使用 Git 管理文档版本
3. 手动保存：生成型内容手动落盘
4. 定期清理：删除过时文档

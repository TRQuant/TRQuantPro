TRQuant-Scripts 工作区工作流规范
================================

核心原则
--------
Composer 仅用于小改（<20行），生成型任务使用 Chat

Composer 使用规则
-----------------
⚠️ 仅用于小改（<20行变更）

可以使用 Composer：
- ✅ 修复 bug（<20行）
- ✅ 小范围重构（<20行）
- ✅ 调整参数（<10行）

禁止使用 Composer：
- ❌ 生成新脚本
- ❌ 大范围修改（>20行）
- ❌ 批量生成脚本

标准流程
--------

脚本生成流程
------------
1. 使用 Chat 模式
2. 让 AI 生成完整脚本内容
3. 手动新建/覆盖文件
4. 保存文件
5. 测试脚本功能

脚本修改流程（小改）
-------------------
1. 使用 Composer（<20行变更）
2. 小范围修改
3. 保存文件
4. 测试脚本功能

脚本修改流程（大改）
-------------------
1. 使用 Chat 模式
2. 说明需要修改的内容
3. 复制 AI 生成的内容
4. 手动编辑文件
5. 保存文件
6. 测试脚本功能

绝对禁止
--------
❌ 让 Composer 生成新脚本
❌ 让 Composer 大范围修改（>20行）
❌ 批量生成脚本
❌ 在 Composer 中一次性生成多个脚本

文件大小建议
------------
- 单个脚本建议 < 500 行
- 大脚本拆分为多个小脚本
- 使用模块化设计

最佳实践
--------
1. 脚本拆分：大脚本拆分为多个小脚本
2. 版本管理：使用 Git 管理脚本版本
3. 手动保存：生成型内容手动落盘
4. 测试验证：生成后立即测试脚本功能

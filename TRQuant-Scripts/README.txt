TRQuant-Scripts 工作区
=====================

工作区说明
----------
这是 TRQuant 脚本生成工作区，专门用于生成型脚本的创建和维护。

工作区用途
----------
- 数据收集脚本生成
- 自动化脚本创建
- 工具脚本开发
- 临时脚本管理

重要规则
--------

Composer 使用规则
-----------------
仅用于小改（<20行变更）

原因：
- 生成型脚本通常从零开始创建
- 大范围修改容易超时
- 脚本文件可能包含复杂逻辑

工作流程
--------
使用 Chat 模式生成：
1. 使用 Chat 生成脚本内容
2. 手动创建/覆盖文件
3. 保存文件
4. 测试脚本功能

Composer 仅用于小改：
- 可以用于修复 bug（<20行）
- 可以用于小范围重构（<20行）
- 禁止用于生成新脚本
- 禁止用于大范围修改（>20行）

目录结构
--------
TRQuant-Scripts/
├── scripts/                 # 所有脚本
│   ├── *.py                 # Python 脚本
│   ├── *.sh                 # Shell 脚本
│   └── ...
├── tools/
│   └── data_collector/      # 数据收集脚本
├── .cursor/                 # Cursor 配置
└── README.txt              # 本文件

配置说明
--------
Cursor 配置：
在 .cursor/workflow_guidelines.txt 中明确：
- Composer: 仅小改（<20行）
- Chat: 可用（生成脚本）
- 文件大小限制: 建议 < 500行

最佳实践
--------
1. 脚本拆分：大脚本（>500行）拆分为多个小脚本
2. 版本管理：使用 Git 管理脚本版本
3. 手动保存：生成型内容手动落盘
4. 测试验证：生成后立即测试脚本功能

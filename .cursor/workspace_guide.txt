TRQuant 工作区使用指南
=====================

工作区划分
----------
1. TRQuant（代码开发）
   路径: /home/taotao/dev/QuantTest/TRQuant/
   用途: 核心代码开发
   Composer: 可用（只做小改 <50行）

2. TRQuant-Docs（文档维护）
   路径: /home/taotao/dev/QuantTest/TRQuant-Docs/
   用途: 文档创建和维护
   Composer: 禁用（只用 Chat）

3. TRQuant-Scripts（脚本生成）
   路径: /home/taotao/dev/QuantTest/TRQuant-Scripts/
   用途: 生成型脚本
   Composer: 仅小改（<20行）

如何切换工作区
--------------
在 Cursor 中：
1. File -> Open Folder
2. 选择对应的工作区路径
3. 开始工作

工作区选择指南
--------------
需要开发代码？
  → 打开 TRQuant 工作区
  → 使用 Composer 做小改（<50行）

需要创建/编辑文档？
  → 打开 TRQuant-Docs 工作区
  → 使用 Chat 生成文档，手动保存

需要生成脚本？
  → 打开 TRQuant-Scripts 工作区
  → 使用 Chat 生成脚本，手动保存
  → 小改（<20行）可用 Composer

已迁移内容
----------
到 TRQuant-Docs:
- docs/ 目录（所有文档）
- extension/AShare-manual/ 目录（A股手册）

到 TRQuant-Scripts:
- scripts/ 目录（所有脚本）
- tools/data_collector/ 目录（数据收集脚本）

主工作区保留:
- core/ 目录（核心代码）
- mcp_servers/ 目录（MCP 服务器）
- gui/ 目录（GUI 代码）
- extension/ 目录（VS Code 扩展，排除 AShare-manual）
- 其他核心代码文件

注意事项
--------
1. 不要在错误的工作区做错误的事情
2. 文档和脚本已迁移，主工作区不再包含这些内容
3. 如果需要访问文档或脚本，切换到对应工作区
4. 保持工作区隔离，避免 Composer 性能问题

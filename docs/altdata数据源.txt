按**个人投资 + 前期控成本**这个约束，下面给你一套**现实、可落地、工程可控**的 **AltData 替代方案**。结论先行，然后给**分层替代表 + 推荐组合 + 实施顺序**。

---

# 一句话结论（非常明确）

> **个人投资阶段，不要上 Wind / 巨灵 / Choice 全家桶。**
> **用「公开数据 + 少量低成本平台 + 自建抽取」即可覆盖 70–80% 的十倍股早期有效信息。**
> 剩下 20–30% 的信息密度，**不是靠花钱解决，而是靠研究方法和证据链设计解决。**

---

# 一、成本约束下的 AltData 分层替代方案

下面按 **“必须有 / 强烈建议 / 可以没有”** 分三档。

---

## ✅ 第一档（必须有，几乎 0 成本）

### 1️⃣ 上市公司公告 / 年报 / 临时公告

**数据源**

* 交易所官网（上交所 / 深交所）
* 巨潮资讯网（cninfo，官方）

**成本**

* 💰 0 元

**工程方式**

* 合规：公开信息
* 技术：

  * 简单爬虫 or RSS
  * 落 `RawDoc`
  * 用规则 + LLM 抽取事件

**十倍股价值**

* 扩产、募投、项目、环评、验证进展
* **可信度最高的证据源**

👉 **这是十倍股体系的“地基”，不需要任何付费数据。**

---

### 2️⃣ 互动易问答（极高性价比）

**数据源**

* 上证 e互动
* 深证互动易

**成本**

* 💰 0 元

**工程方式**

* 定向爬取公司问答
* 抽取关键词：

  * 送样 / 验证 / 客户 / 量产 / 良率 / 产能

**十倍股价值**

* 最早披露“送样 / 验证 / 客户导入”
* 比公告早、比研报真实

👉 **很多十倍股 S1→S2 信号，第一次出现在互动易。**

---

### 3️⃣ 财务数据（聚宽 JQData 个人版）

**数据源**

* JQData（个人量化用户）

**成本**

* 💰 低（年费可控，个人可承受）

**覆盖**

* 三大报表
* 毛利、研发、现金流、应收
* 股东/股本/解禁/行业

**十倍股价值**

* 你列的 5 个“早期财务特征”，**JQData 足够用**

👉 **财务因子这块，不要自己造轮子。**

---

## ✅ 第二档（强烈建议，低成本替代）

### 4️⃣ 招投标 / 项目数据（低成本方案）

**替代方案（不买商业库）**

* 中国招标投标公共服务平台（官方）
* 各省公共资源交易中心网站
* 环评公示网站（地方生态环境局）

**成本**

* 💰 0 元（时间成本）

**工程方式**

* 不做“全量爬”
* 只爬：

  * 特定公司
  * 特定关键词（材料 / 设备 / 工艺）
* 落为 `ProjectEvent`

**十倍股价值**

* 扩产、进入大客户供应链的**硬证据**
* 比财报早 1–3 个季度

👉 **这是“穷人版巨灵项目库”，但信息价值依然很高。**

---

### 5️⃣ 招聘数据（组织信号，低成本）

**替代方案**

* 拉勾 / BOSS / 猎聘 **网页级抓取**
* 只抓：

  * 岗位数量变化
  * 岗位类型（生产 / 工艺 / 销售 / 质量）

**成本**

* 💰 0 元（谨慎抓取频率）

**工程方式**

* 不存原始简历
* 只存：

  * 公司
  * 岗位类别
  * 城市/基地
  * 数量趋势

**十倍股价值**

* 研发 → 量产 → 销售 的组织转折信号
* 扩产前常出现“制造/工艺/质量”岗位激增

👉 **这是判断“是否真在产业化”的重要侧证。**

---

## ⚠️ 第三档（可以没有 / 后期再说）

### 6️⃣ 研报数据库（昂贵，但可被替代）

**不推荐前期购买 Wind / Choice**

**替代方案**

* 东方财富 / 同花顺 研报页面（只统计数量，不看全文）
* 券商公众号（手动/半自动）

**十倍股价值**

* 研报**数量本身**就是指标（共识度）
* 不需要全文，不需要预测表

👉 **十倍股早期 = 研报越少越好。**

---

### 7️⃣ 舆情 / 社媒（可选）

**结论**

* 对十倍股早期识别：**边际价值低**
* 容易制造噪音

**建议**

* 后期只作为：

  * 注意力反向指标
  * 情绪过热预警

---

# 二、个人投资阶段的「最优低成本组合」

### 💡 推荐组合（<= 最小可行系统）

| 模块    | 数据来源         | 成本 |
| ----- | ------------ | -- |
| 公告/年报 | 交易所 / 巨潮     | 0  |
| 互动易   | 上证e互动 / 深证互动 | 0  |
| 财务    | JQData       | 低  |
| 招投标   | 官方网站定向抓取     | 0  |
| 招聘    | 招聘网站趋势抓取     | 0  |
| 新闻    | 财联社/公开媒体     | 0  |

👉 **这套组合 ≈ 商业 AltData 的 70% 信息密度**

---

# 三、在韬睿系统里怎么“工程化”这套低成本方案

## 1️⃣ AltDataAdapter（穷人版）

* 只支持：

  * HTTP 抓取
  * RSS
  * 文件导入
* 统一落 `RawDoc`
* 每条数据带：

  * 来源
  * 时间
  * 可信度权重

---

## 2️⃣ Event 抽取比“数据源”更重要

你要把钱和精力花在 **Event 层**，不是 Data 层。

### 十倍股最有价值的事件类型（V1 就够）

* 送样 / 验证
* 进入客户体系
* 小批量 / 重复订单
* 扩产 / 新产线 / 环评
* 招聘结构变化
* 高管 / 组织补强

**V1 实现**

* 关键词 + 正则
* LLM 辅助分类
* 人工纠错入口

---

## 3️⃣ 用“证据链”抵消数据不全

你前面设计的逻辑本身就是为“信息不完备”而生的：

> **单条证据不可靠 → 多证据交叉验证 → 状态机推进**

* 公告 + 互动易 + 招聘 + 项目
  → 置信度累加
  → Stage 从 S1 → S2 → S3

👉 **这是个人投资者对抗机构数据优势的唯一正确姿势。**

---

# 四、个人阶段不该做的 3 件事（非常重要）

❌ 一开始就追求“全市场 + 全数据”
❌ 为了“像机构”而买 Wind
❌ 在 Data 层花太多时间，而忽略 Event / Stage / Thesis

---

# 五、建议的落地顺序（极其重要）

**第 1 步（马上能做）**

* 公告 + 互动易 → RawDoc → Event → Stage
* 跑 1 个行业（比如半导体材料）

**第 2 步**

* 接入 JQData 财务
* 上 7 维评分卡 V1

**第 3 步**

* 加入招投标 + 招聘（只做趋势，不做全量）

**第 4 步**

* 再考虑是否值得买商业 AltData

---

# 六、最终判断（给你一个底线）

> **个人投资阶段，十倍股早期识别的瓶颈从来不是“有没有 Wind”，
> 而是：
> 你能不能把零散、公开、低频的信息，
> 变成“有方向、有节奏、有证伪机制”的证据链系统。**

你现在这套 **“产业位置 + 兑现路径 + 财务拐点 + 估值错配”** 的方法论，**非常适合低成本数据环境**，反而不适合“数据堆砌”。
